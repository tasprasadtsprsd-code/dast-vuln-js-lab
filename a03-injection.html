A03:2021 · INJECTION

Injection

Injection flaws occur when untrusted data is sent to an interpreter as part of a command or query. Includes SQL injection, XSS (HTML injection), command injection, and template injection.

CWE References
CWE-79, CWE-89, CWE-77
Impact
Data exfiltration, account takeover, RCE

Demo 1 — Reflected XSS via innerHTML

The search query is reflected into the DOM using innerHTML with no sanitization. Any HTML/script payload in the query string executes in the user's browser with full page privileges.

Results appear here…

✅ Remediation

Use textContent for plain strings; DOMPurify when HTML is needed

// VULNERABLE
resultDiv.innerHTML = 'Courses matching: ' + userQuery;

// FIXED — textContent never parses HTML
resultDiv.textContent = 'Courses matching: ' + userQuery;

// FIXED — when HTML output is genuinely required
import DOMPurify from 'dompurify';
resultDiv.innerHTML = DOMPurify.sanitize(userQuery);

// Also add Content-Security-Policy header:
// Content-Security-Policy: default-src 'self'; script-src 'self'
  • CSP is the most effective second layer — it stops inline script even if injection occurs.
  • Encode output for the context: HTML encoding for HTML, JS encoding for script blocks.
  • Never build HTML by string concatenation with user data.

Demo 2 — SQL Injection via String Concatenation

The grade lookup query concatenates the student name directly into SQL. The payload ' OR '1'='1 bypasses the WHERE clause and dumps all records.

Payloads: ' OR '1'='1 · '; DROP TABLE grades;--
—

✅ Remediation

Use parameterized queries — never interpolate user input into SQL

// VULNERABLE
const q = `SELECT * FROM grades WHERE name = '${input}'`;
db.query(q);

// FIXED — parameterized (Node postgres / mysql2)
const q = 'SELECT * FROM grades WHERE name = $1';
const { rows } = await db.query(q, [input]);

// FIXED — ORM (Prisma) parameterizes automatically
const grades = await prisma.grade.findMany({
  where: { studentName: input }  // safe — no SQL injection possible
});
  • Parameterized queries guarantee the DB treats input as data, never as SQL syntax.
  • ORMs (Prisma, Sequelize, Drizzle) parameterize by default — prefer them.
  • Never use string formatting, template literals, or concatenation to build SQL.
