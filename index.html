How to use this lab

Each page covers one OWASP Top 10 category. Start with the Vulnerable demo tab to trigger the flaw yourself, then switch to the Remediation tab to see the secure fix with working code.

Point a DAST scanner (OWASP ZAP, Burp Suite) at these pages and compare its findings to the known list — then apply the fixes and re-scan to confirm they disappear.

A01:2021
Broken Access Control
Users act outside their intended permissions — IDOR, privilege escalation, missing auth checks.
CRITICAL
A02:2021
Cryptographic Failures
Sensitive data exposed due to weak or missing encryption — cleartext storage, MD5 hashing.
CRITICAL
A03:2021
Injection
XSS and SQL injection via unsanitized user input written into queries or the DOM.
CRITICAL
A04:2021
Insecure Design
Design flaws — unlimited login attempts, client-side business logic, missing rate limits.
HIGH
A05:2021
Security Misconfiguration
Missing security headers, verbose error messages, default credentials left in place.
HIGH
A06:2021
Vulnerable Components
Using outdated libraries with known CVEs. Prototype pollution, ReDoS, and version exposure.
HIGH
A07:2021
Auth Failures
Weak session tokens, no logout, no account lockout, credentials in localStorage.
CRITICAL
A08:2021
Software Integrity Failures
Loading CDN scripts without SRI hashes, insecure deserialization of JSON payloads.
HIGH
A09:2021
Logging Failures
No logging of auth events, sensitive data in logs, log injection via user input.
MEDIUM
A10:2021
SSRF
Server fetches attacker-supplied URLs — reaching internal services and cloud metadata.
CRITICAL
