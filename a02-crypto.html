A02:2021 · CRYPTOGRAPHIC FAILURES

Cryptographic Failures

Formerly "Sensitive Data Exposure". Failures related to cryptography that lead to exposure of sensitive data — passwords stored in plaintext or with weak hashes like MD5, data transmitted over HTTP, weak encryption keys.

CWE References
CWE-259, CWE-327, CWE-331
Impact
Credential theft, data breach, identity fraud

Demo 1 — Plaintext & MD5-hashed Passwords

The student database stores passwords in plaintext and as MD5 hashes. MD5 is cryptographically broken — any MD5 hash can be cracked in seconds via rainbow tables.

MD5 of "hunter2" → 2ab96390c7dbe3439de74d0c9b0b1767 — crackable instantly
—

✅ Remediation

Use bcrypt, scrypt, or Argon2 — never MD5/SHA1/SHA256 alone for passwords

// VULNERABLE — plaintext or fast hash
db.save({ password: userPassword });
db.save({ password: md5(userPassword) });

// FIXED — bcrypt with work factor ≥ 12 (Node.js)
const bcrypt = require('bcrypt');
const SALT_ROUNDS = 12;

// On registration:
const hash = await bcrypt.hash(plainPassword, SALT_ROUNDS);
db.save({ passwordHash: hash });

// On login:
const match = await bcrypt.compare(inputPassword, stored.passwordHash);
if (!match) return res.status(401).json({ error: 'Invalid credentials' });
  • Never store passwords in plaintext — ever.
  • MD5 and SHA-1 are broken for password hashing — they're too fast.
  • bcrypt's work factor makes brute-force computationally expensive.
  • Use Argon2id where possible — it's the current OWASP recommendation.

Demo 2 — Sensitive Data in LocalStorage (Cleartext)

Student exam scores, SSNs, and session tokens are stored in localStorage without encryption. Any XSS payload or browser extension can read them instantly.

Open DevTools → Application → Local Storage to see all values
—

✅ Remediation

Keep secrets server-side in HttpOnly cookies — never in localStorage

// VULNERABLE — sensitive data in localStorage
localStorage.setItem('student_ssn', '555-01-2345');
localStorage.setItem('exam_scores', JSON.stringify(scores));
localStorage.setItem('auth_token', jwtToken);

// FIXED — session token in HttpOnly cookie (server sets it)
res.cookie('session', token, {
  httpOnly: true,  // JS cannot read this
  secure: true,    // HTTPS only
  sameSite: 'strict',
  maxAge: 3600000
});

// Non-sensitive UI state only goes in localStorage
localStorage.setItem('theme', 'dark');   // safe
localStorage.setItem('lang', 'en');      // safe
  • Treat localStorage as publicly readable — any JS on the page can access it.
  • Never store PII, SSNs, or financial data in client-side storage.
  • Encrypt sensitive data at rest on the server; never on the client.
