A01:2021 · #1 MOST CRITICAL

Broken Access Control

Access control enforces policy so users cannot act outside their intended permissions. Failures lead to unauthorized information disclosure, modification, or destruction of data, or performing business functions outside the user's limits.

CWE References
CWE-200, CWE-284, CWE-285
Impact
Data breach, privilege escalation, account takeover

Demo 1 — IDOR (Insecure Direct Object Reference)

You are logged in as Student #3. The API returns any student record by ID without checking that the requester owns it. Change the ID to access other accounts including the admin.

Try ID 1 (admin) or 2 to access other accounts
—

✅ Remediation

Enforce authorization server-side — never trust a client-supplied ID alone

// VULNERABLE
app.get('/api/students/:id', (req, res) => {
  res.json(db.students.find(s => s.id === +req.params.id));
});

// FIXED — verify session owns the requested resource
app.get('/api/students/:id', authenticate, (req, res) => {
  const id = Number(req.params.id);
  if (id !== req.session.studentId && !req.session.isAdmin) {
    return res.status(403).json({ error: 'Forbidden' });
  }
  const { password, ...safe } = db.students.find(s => s.id === id);
  res.json(safe); // never return password field
});
  • Use UUIDs instead of sequential integers — harder to enumerate.
  • Apply authorization checks at every API layer, not just the UI.
  • Log all 403 responses — repeated forbidden access is an attack signal.

Demo 2 — Privilege Escalation via Client-Side Role

The admin flag is stored in localStorage and read client-side to show/hide the admin panel. An attacker simply sets it to true in the browser console.

—
🔓 ADMIN PANEL UNLOCKED — Delete all records · Export all data · Manage users

✅ Remediation

Roles and permissions must be verified server-side on every request

// VULNERABLE — role checked client-side
const isAdmin = localStorage.getItem('role') === 'admin';
if (isAdmin) showAdminPanel();

// FIXED — role comes from a server-verified session
// Client just makes the request; server decides the response
fetch('/api/admin/users', { credentials: 'include' })
  .then(r => r.status === 200 ? r.json() : Promise.reject(r.status))
  .then(data => renderAdminPanel(data))
  .catch(status => { if (status === 403) hideAdminPanel(); });

// Server:
app.get('/api/admin/users', authenticate, requireRole('admin'), (req, res) => {
  res.json(db.users);
});
  • Never use localStorage/sessionStorage to store roles or permissions.
  • Re-validate role on every sensitive API call — sessions can change.
